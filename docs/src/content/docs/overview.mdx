eyeVesa is an identity and trust layer for the agentic economy. It helps teams answer a simple operational question before an AI agent touches a protected resource: who is this agent, what is it allowed to do, and how do we prove what happened later?

## Platform Responsibilities

- Register agents and issue cryptographic identities.
- Authorize actions through policy decisions and trust scores.
- Proxy MCP tool calls through a gateway.
- Record signed audit events for non-repudiation.
- Discover agents through Airport profiles and heartbeats.
- Escalate high-risk actions through human-in-the-loop approvals.

## Main Packages

| Package | Purpose |
| --- | --- |
| `gateway/core` | Rust MCP proxy, crypto, mTLS termination, and PTV identity checks. |
| `gateway/control-plane` | Go HTTP and gRPC APIs for registration, authorization, policy, audit, Airport, and HITL. |
| `sdk/agent-sdk-python` | Python client library with major agent framework integrations. |
| `sdk/agent-sdk-typescript` | TypeScript client library for Node.js agents. |
| `adapter/resource-adapter-go` | Go MCP server wrapper for enterprise resources. |
| `cli/` | `eyevesa` CLI for local operations and Airport commands. |

## Protocols

eyeVesa combines KYA, Airport, and MCP:

- KYA identifies and authorizes the agent.
- Airport lets agents discover each other.
- MCP handles tool execution through registered resources.
