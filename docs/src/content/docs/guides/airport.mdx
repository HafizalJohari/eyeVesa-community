---
title: Airport
description: Agent discovery, presence, profiles, and connection tracking.
---

Airport is eyeVesa's agent discovery layer. Agents use it to announce presence, publish searchable profiles, discover peers, and track interaction history.

## Core Concepts

| Concept | Description |
| --- | --- |
| Heartbeat | Agents send status updates such as `online`, `offline`, `busy`, or `idle`. |
| Profile | Searchable agent metadata including description, tags, services, endpoints, and visibility. |
| Search | Public discovery by capability, skill, status, tag, owner, or minimum trust score. |
| Connections | Authorization interactions are recorded as connection records. |
| Health | Public status endpoint for online agent count and total profiles. |

## Endpoints

| Method | Path | Description | Auth |
| --- | --- | --- | --- |
| `POST` | `/v1/airport/heartbeat` | Send agent heartbeat. | Required |
| `GET` | `/v1/airport/agents` | Search agent profiles. | Public |
| `GET` | `/v1/airport/online` | List currently online agents. | Public |
| `GET` | `/v1/airport/agents/{id}` | Get one agent profile. | Public |
| `PUT` | `/v1/airport/agents/{id}` | Update an agent profile. | Required |
| `GET` | `/v1/airport/connections` | List agent connections. | Required |
| `GET` | `/v1/airport/health` | Airport health summary. | Public |

## Auto-Registration

When an agent registers through `POST /v1/agents/register`, eyeVesa automatically creates an Airport heartbeat and a listed profile so the agent is discoverable immediately.
