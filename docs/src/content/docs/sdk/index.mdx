## Common Agent Lifecycle

<Steps>
1. Create an `AgentConfig` with identity, owner, and gateway endpoint.
2. Connect to the gateway. The SDK registers the agent and performs login.
3. Discover MCP-capable resources.
4. Invoke tools. The gateway authorizes the action before execution.
5. Use Airport to publish presence and discover peers.
6. For sensitive workflows, issue capability tokens and signed receipts.
</Steps>

## Quick Compare

<Tabs>
  <TabItem label="TypeScript">

```ts
import { AgentClient } from 'agentid-sdk';

const client = new AgentClient({
  agentId: 'ops-agent-001',
  name: 'ops-agent',
  owner: 'platform-team',
  gatewayEndpoint: 'http://localhost:9443',
});

await client.connect();
const tools = await client.discover('mcp');
const result = await client.invoke(tools[0].resourceId, 'read', {
  location: 'Kuala Lumpur',
});
```

  </TabItem>
  <TabItem label="Python">

```python
from agentid_sdk import AgentClient, AgentConfig

config = AgentConfig(
    agent_id="ops-agent-001",
    name="ops-agent",
    owner="platform-team",
    gateway_endpoint="http://localhost:9443",
)

async with AgentClient(config) as client:
    await client.connect()
    tools = await client.discover("mcp")
    result = await client.invoke(
        resource_id=tools[0].resource_id,
        tool="read",
        params={"location": "Kuala Lumpur"},
    )
```

## Authentication Modes

| Mode | Use When | SDK Shape |
| --- | --- | --- |
| Registration + login | Local dev and agent-owned key flows. | `await client.connect()` |
| API key | Production gateway with `AUTH_ENABLED=true`. | `new AgentClient(config, { apiKey })` |
| JWT | Existing identity provider or gateway-issued token. | `new AgentClient(config, { jwtToken })` |
| Environment config | Runtime config is injected by deployment. | `AgentClient.fromEnv()` / `AgentClient.from_env()` |
