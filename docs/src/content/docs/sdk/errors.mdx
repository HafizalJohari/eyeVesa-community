SDKs expose specific errors so agent runtimes can distinguish gateway connectivity, policy denial, HITL escalation, MCP failures, and transaction verification problems.

## Exception Hierarchy

```text
AgentIDError
  +-- ConnectError
  |     +-- AuthFailedError
  +-- DiscoverError
  +-- InvokeError
  |     +-- NotAuthorizedError
  |     +-- HitlRequiredError
  +-- DelegateError
  |     +-- MaxDepthError
  +-- PtvError
  +-- HitlError
  +-- McpError
  +-- VerifyError
  +-- SkillError
  +-- TxError
```

## Handling Policy Outcomes

<Tabs>
  <TabItem label="TypeScript">

```ts
import {
  HitlRequiredError,
  NotAuthorizedError,
} from 'agentid-sdk';

try {
  await client.invoke(resourceId, 'delete', { id: 'invoice-123' });
} catch (error) {
  if (error instanceof HitlRequiredError) {
    await notifyApprover(error.message);
  } else if (error instanceof NotAuthorizedError) {
    await recordDeniedAction(error.message);
  } else {
    throw error;
  }
}
```

## Recommended Recovery

| Error | Meaning | Recovery |
| --- | --- | --- |
| `ConnectError` | Registration, heartbeat, or profile update failed. | Check gateway URL, auth, network, and service health. |
| `AuthFailedError` | API key, JWT, or signed login failed. | Refresh credentials or re-register identity. |
| `DiscoverError` | No resource or Airport result could be found. | Retry with broader filters or confirm resources are registered. |
| `NotAuthorizedError` | Policy denied the action. | Do not retry blindly; surface the policy reason. |
| `HitlRequiredError` | Policy requires human approval. | Create or monitor a HITL approval flow. |
| `McpError` | MCP initialization, listing, or tool call failed. | Inspect MCP adapter health and request payload. |
| `TxError` | Token or receipt operation failed. | Verify token status, expiry, revocation state, and receipt signature. |

## Logging Checklist

- Agent ID and owner.
- Resource ID and action.
- Policy reason or HTTP status.
- Trust score and trust delta when available.
- HITL approval ID when escalation happens.
- Token ID and receipt ID for transaction flows.
