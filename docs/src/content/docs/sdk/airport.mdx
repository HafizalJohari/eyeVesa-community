Airport is the discovery layer for agents. SDK helpers wrap the `/v1/airport/*` endpoints so agents can stay online, publish profiles, search peers, and inspect connection history.

## Presence Flow

<Steps>
1. Connect the agent to establish identity.
2. Send an Airport heartbeat.
3. Publish or update the agent profile.
4. Search for peers by capability, skill, trust score, status, tag, or owner.
5. Query connection history for audit or graph workflows.
</Steps>

## Heartbeat

<Tabs>
  <TabItem label="TypeScript">

```ts
await client.airportHeartbeat('online', {
  runtime: 'node',
  region: 'local-dev',
});
```

  </TabItem>
  <TabItem label="Python">

```python
await client.airport_heartbeat(
    status="online",
    metadata={"runtime": "python", "region": "local-dev"},
)
```

  </TabItem>
</Tabs>

Valid statuses are typically `online`, `offline`, `busy`, and `idle`.

## Profile Update

<Tabs>
  <TabItem label="TypeScript">

```ts
await client.airportUpdateProfile({
  description: 'Customer support agent for ticket triage and CRM lookup.',
  servicesOffered: ['ticket_triage', 'crm_lookup'],
  endpoints: {
    mcp: 'http://localhost:9443/v1/mcp',
  },
  tags: ['support', 'crm', 'trusted'],
  listed: true,
});
```

  </TabItem>
  <TabItem label="Python">

```python
await client.airport_update_profile(
    description="Customer support agent for ticket triage and CRM lookup.",
    services_offered=["ticket_triage", "crm_lookup"],
    endpoints={"mcp": "http://localhost:9443/v1/mcp"},
    tags=["support", "crm", "trusted"],
    listed=True,

```

  </TabItem>
</Tabs>

## Endpoint Map

| SDK helper | Endpoint | Auth |
| --- | --- | --- |
| `airportHeartbeat()` / `airport_heartbeat()` | `POST /v1/airport/heartbeat` | Required |
| `airportUpdateProfile()` / `airport_update_profile()` | `PUT /v1/airport/agents/{id}` | Required |
| `airportSearch()` / `airport_search()` | `GET /v1/airport/agents` | Public |
| `airportGetProfile()` / `airport_get_profile()` | `GET /v1/airport/agents/{id}` | Public |
| `airportListOnline()` / `airport_list_online()` | `GET /v1/airport/online` | Public |
| `airportConnections()` / `airport_connections()` | `GET /v1/airport/connections` | Required |

<Aside type="caution">
Keep write operations authenticated in production. Public search is useful for discovery, but heartbeat, profile mutation, and connection history should be protected by API key or JWT auth.
</Aside>
