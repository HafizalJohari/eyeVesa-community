The TypeScript SDK is the Node.js client for agents that need eyeVesa identity, authorization, MCP tool execution, Airport discovery, and transaction receipts.

## Install

```bash
npm install agentid-sdk
```

Requirements:

- Node.js 18 or newer.
- A running eyeVesa gateway.
- `AUTH_ENABLED=true` gateways require either an API key, JWT token, or a successful `connect()` login flow.

## Minimal Client

```ts
import { AgentClient, type AgentConfig } from 'agentid-sdk';

const config: AgentConfig = {
  agentId: 'support-agent-001',
  name: 'support-agent',
  owner: 'customer-success',
  gatewayEndpoint: 'http://localhost:9443',
};

const client = new AgentClient(config);
await client.connect();

console.log(client.agentId);
console.log(client.trustScore);
```

## Environment Setup

```bash
export AGENT_ID=support-agent-001
export AGENT_NAME=support-agent
export AGENT_OWNER=customer-success
export GATEWAY_ENDPOINT=http://localhost:9443
```

```ts
import { AgentClient } from 'agentid-sdk';

const client = AgentClient.fromEnv();
await client.connect();
```

<Aside type="note">
`AgentClient.fromEnv()` falls back to a generated agent ID, `node-agent`, `default`, and `http://localhost:8080` when variables are missing. Set the variables explicitly for repeatable local testing.
</Aside>

## Invoke A Resource

<Steps>
1. Connect the agent.
2. Discover resources by capability.
3. Choose the target resource ID.
4. Invoke the tool. The SDK requests authorization before MCP execution.
</Steps>

```ts
const tools = await client.discover('mcp');
const weatherTool = tools.find((tool) => tool.name.includes('weather'));

if (!weatherTool) {
  throw new Error('No weather MCP resource is registered');
}

const result = await client.invoke(weatherTool.resourceId, 'read', {
  location: 'Kuala Lumpur',
});

console.log(result.success);
console.log(result.trustScore);
console.log(result.data);
```

## Core API

| Method | Returns | Purpose |
| --- | --- | --- |
| `connect()` | `Promise<AgentClient>` | Register agent, sign login challenge, and update JWT state. |
| `register()` | `Promise<AgentClient>` | Register agent identity and public key only. |
| `login()` | `Promise<AgentClient>` | Sign a nonce and receive a JWT token. |
| `discover(capability)` | `Promise<ToolInfo[]>` | Discover resources matching a capability such as `mcp`. |
| `invoke(resourceId, tool, params)` | `Promise<InvokeResult>` | Authorize then execute a tool through MCP. |
| `delegate(delegateeId, scope, reason)` | `Promise<DelegateResult>` | Delegate scoped permissions to another agent. |
| `verifySignature(agentId, message, signature)` | `Promise<boolean>` | Verify an Ed25519 agent signature. |
| `mcpInitialize()` | `Promise<McpCapabilities>` | Initialize direct MCP capability negotiation. |
| `mcpListTools()` | `Promise<McpTool[]>` | List MCP tools exposed by the gateway. |
| `mcpCallTool(toolName, arguments)` | `Promise<Record<string, unknown>>` | Call an MCP tool directly. |

interface AgentConfig {

## Result Types

```ts
interface ToolInfo {
  name: string;
  description: string;
  resourceId: string;
  parameters: Record<string, unknown>;
}

interface InvokeResult {
  success: boolean;
  data: Record<string, unknown>;
  trustScore: number;
}
```

## Error Handling

```ts
import {
  HitlRequiredError,
  NotAuthorizedError,
} from 'agentid-sdk';

try {
  await client.invoke(resourceId, 'delete', { id: 'invoice-123' });
} catch (error) {
  if (error instanceof HitlRequiredError) {
    console.log('Human approval required:', error.message);
  } else if (error instanceof NotAuthorizedError) {
    console.log('Policy denied:', error.message);
  } else {
    throw error;
  }
}
```

See [Errors](/sdk/errors/) for the full exception hierarchy.
